### 자료구조
#### 자료구조란?
자료구조란 여러 데이터의 묶을을 저장하고 사용하는 방법을 정의한 것


자료구조란 데이터를 체계적으로 정리하고 저장하여 수많은 상황에 맞게 데이터를 효율적으로 다룰 수 있는 방법들을
일컫는 말이다.


대부분의 자료구조는 특정한 상황에 놓인 문제를 해결하는 데 특화되어 있다.<br>
따라서 많은 자료구조를 알아두면 어떤 상황이 닥쳤을 때 적합한 자료구조를 적용하여 문제를 원활하게 해결할 수 있다.


자주 등장하는 자료 구조로는 Stack Queue, Tree, Graph가 있다.

#### Stack
스택(Stack)은 '쌓다', '포개지다'와 같은 뜻을 지닌 단어로 데이터를 순서대로 쌓는 자료구조이다.<br>
대표적으로 브라우저의 뒤로 가기, 앞으로 가기 기능을 구현할 때 활용된다.


##### Stack의 특징
1. LIFO(Last In First Out)
: 먼저 들어간 데이터가 가장 나중에 나오는 후입선출의 구조를 지닌다
2. 데이터는 하나씩 넣고 뺄 수 있다
: 많은 양의 데이터가 있어도 한꺼번에 여러 데이터를 넣거나 뺄 수 없다
3. 하나의 입출력 방향을 가지고 있다
: Stack 자료구조는 데이터의 입출력 뱡향이 같다

#### Queue
큐(Queue)sms '줄을 서서 기다리다', '대기행렬'이란 뜻은 지닌 단어로 데이터를 순서대로 처리하는 자료구조이다.<br>
대표적으로 프린터의 인쇄 기능을 구현할 때 사용된다.

##### Queue의 특징
1. FIFO(First In First Out)
: 먼저 들어간 데이터가 가장 먼저 나오는 선입선출의 구조를 지닌다
2. 데이터를 하나씩 넣고 뺄 수 있다
: 많은 양의 데이터가 있어도 한꺼번에 여러 데이터를 넣거나 뺄 수 없다
3. 두 개의 입출력 방향을 가지고 있다
: Queue 자료구조는 데이터의 입력과 출력 방향이 다르다
