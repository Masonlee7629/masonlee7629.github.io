## 자료구조
### Tree
Tree는 이름 그대로 나무의 형태를 가지고 있다.<br>
하나의 뿌리로부터 가지가 사방으로 뻗은 형태가 나무와 닮아있다고 해서 트리 구조라고 부른다.


트리 구조는 데이터가 바로 아래에 있는 하나 이상의 데이터에 무방향으로 연결된 계층적 자료구조이다.<br>
하나의 데이터 아래에 여러 개의 데이터가 존재할 수 있는 비선형 구조로 아래로만 뻗어나가기 때문에 사이클이 없다.

![image](https://user-images.githubusercontent.com/105192204/192151543-3758f3a4-9f67-4e92-afaa-db8225d8c4b5.png){: width="70%", height="70%"}


#### Tree의 특징
트리 구조는 루트(Root)라는 하나의 꼭짓점 데이터를 시작으로 여러 개의 데이터를 간선(edge)로 연결된 것이다.<br>
각 데이터는 노드(Node)라고 하며 두 개의 노드가 계층적으로 연결되어 부모/자식관계를 가진다.<br>
상위 계층의 노드는 부모 노드(Parent Node)이며 자식 계층의 노드는 자식 노드(Child Node)이다.<br>
자식 노드가 없는 노드는 리프 노드(Leaf Node)라고 부른다.

#### Tree의 용어
노드(Node)
: 트리 구조를 이루는 모든 개별 데이터

루트(Root)
: 트리 구조의 시작점이 되는 노드

부모 노드(Parent Node)
: 두 노드가 상하관계로 연결되어 있을 때 상대적으로 루트에서 가까운 노드

자식 노드(Child Node)
: 두 노드가 상하관계로 연결되어 있을 때 상대적으로 루트에서 먼 노드

리프 노드(Leaf Node)
: 트리 구조의 끝 지점으로 자식 노드가 없는 노드

깊이(Depth)
: 루트에서 하위 계층의 특정 노드까지의 깊이. 루트의 깊이는 0이며 하위 계층일수록 1씩 추가

레벨(Level)
: 같은 깊이를 가지고 있는 노드를 표현한 것. 깊이가 0인 루트의 level은 1이며 하위 계층일수록 1씩 추가

높이(Height)
: 리프 노드를 기준으로 루트까지의 높이를 표현한 것. 높이를 표현할 때 각 리프 노드의 높이를 0으로 지정

서브 트리(Sub Tree)
: 루트에서 뻗어 나오는 큰 트리의 내부에 트리 구조를 갖춘 작은 트리

### Graph
그래프는 여러 개의 점들이 서로 복잡하게 연결되어 있는 관계를 표현한 자료구조이다.<br>
컴퓨터 공학에서 이야기하는 자료구조 그래프는 일반적인 수학적 그래프와 다르다.<br>
자료구조의 그래프는 거미줄처럼 여러 개의 점들이 선으로 이어져 있는 복잡한 네트워크망과 같은 모습을 보인다.
<br>
<br>

![image](https://user-images.githubusercontent.com/105192204/192152763-139eba0b-7126-4c9b-8403-7c02479e16a4.png){: width="60%", height="60%"}

#### Graph의 특징
그래프에서 하나의 점을 정점(vertex)라고 하며 선을 간선(edge)이라 한다.<br>
그래프는 여러 정점과 정점들의 관계를 나타내는 간선으로 이루어진 구조이다.

#### Graph의 표현 방식
인접 행렬
: 서로 다른 정점들이 인접한 상태인지 표시한 행렬로 2차원 배열의 형태로 표현한 것
: 정점이 이어져 있따면 1(true), 이어져 있지 않다면 0(false)으로 표시한 표의 형태
: 두 정점 사이에 관계의 유무를 확인하기에 용이함
: 가장 빠른 경로(shortest path)를 찾을 때 주로 사용

인접 리스트
: 각 정점이 어떤 정점과 인접하는지 리스트의 형태로 표현한 것
: 각 정점은 하나의 리스트를 가지며 자신과 인접한 다른 정점을 담고 있음
: 인접 리스트의 순서는 보통 중요하지 않으나 우선 순위를 고려하여 정렬이 가능
: 메모리를 효율적으로 사용하고 싶을 때 사용

#### Graph의 용어
정점(vertex)
: 노드(node)라고도 하며 데이터가 저장되는 그래프의 기본 원소

간선(edge)
: 정잠 간의 관계를 나타내는 선

인접 정점(adjacent vertex)
: 하나의 정점에서 간선에 의해 직접 연결되어 있는 정점

가중치 그래프(weighted Graph)
: 연결의 강도(추가적인 정보)가 얼마나 되는지 적혀져 있는 그래프

비가중치 그래프(unweighted Graph)
: 연결의 강도가 적혀져 있지 않는 그래프

무(방)향 그래프(undirected Graph)
: 간선의 방향성이 없는 그래프로 각 간선이 양방향으로 진행할 수 있음

단방향 그래프(directed Graph)
: 간선의 방향성이 있는 그래프로 각 간선이 한 방향으로만 진행할 수 있음

진입차수(in-degree) / 진출차수(out-degree)
: 한 정점에 진입(들어오는 간선)하고 진출(나가는 간선)하는 간선이 몇 개인지 나타냄

자기 루프(self loop)
: 정점에서 진출하는 간선이 곧바로 자기 자신에게 진입하는 경우를 표현한 것
: 다른 정점을 거치지 않음

사이클(cycle)
: 한 정점에서 출발하여 다시 해당 정점으로 돌아갈 수 있을 때 사이클이 있다고 표현

### BST(Binary Search Tree)
이진 탐색 트리는 트리 구조 중 가장 간단하고 많이 사용하는 트리 구조이다.

이진 트리(Binary Tree)는 자식 노드가 최대 두 개인 노드들로 구성된 트리이다.<br>
이 두 개의 자식 노드는 왼쪽 자식 노드와 오른쪽 자식 노드로 나눌 수 있다.

|이진 트리 종류|설명|
|:---:|:---:|
|정 이진 트리(Full Binary Tree)|각 노드가 0개 혹은 2개의 자식 노드를 가진 트리|
|포화 이진 트리(Perfect Binary Tree)|정 이진 트리이며 완전 이진 트리인 경우. 모든 리프 노드의 레벨이 동일하고 모들 레벨이 가득 채워진 트리|
|완전 이진 트리(Complete Binary Tree)|마지막 레벨을 제외한 모든 노드가 가득 차 있어야 하며 마지막 레벨의 노드의 왼쪽이 채워진 트리|

이진 탐색 트리는 이진 트리와 구조적으로 비슷하나 다음과 같은 특수한 규칙을 지닌다.
- 모든 왼쪽 자식의 값이 루트나 부모보다 작다
- 모든 오른쪽 자식의 값이 루트나 부모보다 크다
