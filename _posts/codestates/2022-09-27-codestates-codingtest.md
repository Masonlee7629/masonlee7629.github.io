## 코딩 테스트 준비
### 의사코드(pseudocode)
의사코드란 프로그래밍 언어로 코드를 작성하기 전에 우리가 쓰는 일상 언어로 프로그램이 작동하는 논리를 먼저 작성하는 것이다.

의사코드를 작성하면 다음과 같은 장점이 있다.
1. 시간이 단축된다
2. 디버깅에 용이하다
3. 프로그래밍 언어를 모르는 사람과 소통할 수 있다

의사코드는 대표적으로 두 가지 방식이 사용된다.
1. 다른 사람도 이해할 수 있는 자연어만 사용한다
2. 자연어와 프로그램 언어의 조합을 사용한다

### 시간 복잡도(Time Complexity)
알고리즘의 시간복잡도는 입력을 나타내는 문자열 길이의 함수로서 작동하는 알고리즘을 취해 시간을 정량화하는 것이다.<br>
이는 알고리즘에서 효율적인 방법을 고민한다는 것과 동일하다.<br>

문제 해결을 위한 알고리즘을 코드로 구현할 때 시간 복잡도를 고려한다는 것은 다음을 의미한다.
- 입력값의 변화에 따라 연산을 실행할 때 연산 횟수에 비해 시간이 얼마만큼 걸리는가

즉, 효율적인 알고리즘을 구현한다는 것은 '입력값이 커짐에 따라 증가하는 시간의 비율을 최소화한 알고리즘'을 구현했다는 것이다.<br>

![image](https://user-images.githubusercontent.com/105192204/192529014-f6f9967d-6b21-4cf6-b40c-0927bd8e1bef.png)

#### 시간 복잡도 표기법
Big-O(빅-오) : 최악의 경우에 대해 나타내는 방법<br>
Big-Ω(빅-오메가) : 최선의 경우에 대해 나타내는 방법<br>
Big-θ(빅-세타) : 중간(평균)의 경우에 대해 나타내는 방법


위 표기법 중 Big-O 표기법이 가장 자주 사용된다.<br>
프로그램이 실행되는 과정에서 소요되는 최악의 시간까지 구려하여 그에 맞는 대응이 가능하기 때문이다.

#### Big-O 표기법
O(1) (constant complexity)
: 입력값이 증가하여도 시간이 늘어나지 않는 것을 의미한다

O(n) (linear complexity)
: 입력값이 증가함에 따라 시간 또한 같은 비율로 증가하는 것을 의미한다

O(log n) (logarithmic complexity)
: Big-O 표기법 중 O(1) 다음으로 빠른 시간 복잡도를 가진다
: 자료구조에서 학습한 BST의 값 탐색도 같은 로직으로 O(log n)d의 시간 복잡도를 가진 알고리즘이다

O(n<sup>2</sup>) (quadratic complexity)
: 입력값이 증가함에 따라 시간이 n의 제곱수의 비율로 증가하는 것을 의미한다

O(2<sup>n</sup>) (exponential complexity)
: Big-O 표기법 중 가장 느린 시간 복잡도를 가진다
: 대표적으로 재귀로 구현하는 피보나치 수열이 있다

### 탐욕 알고리즘(Greedy)
탐욕 알고리즘은 선택의 순간마다 당장 눈앞에 보이는 최적의 상황만을 쫓아 최종적인 해답에 도달하는 방법이다.


탐욕 알고리즘을 이용한 문제 해결 방법의 단계는 다음과 같다.
1. 선택 절차(Selection Procedure)
: 현재 상태에서의 최적의 해답을 선택한다
2. 적절성 검사(Feasibility Check)
: 선택된 해가 문제의 조건을 만족하는지 검사한다
3. 해답 검사(Solution Check)
: 원래의 문제가 해결되었는지 검사하고 해결되지 않았다면 선택 절차로 돌아가 과정을 반복한다

탐욕 알고리즘을 적용하기 위해선 해결하려는 문제가 다음의 2가지 조건을 성립해야한다.
1. 탐욕적 선택 속성(Greedy Choice Property)
: 앞의 선택이 이후의 선택에 영향을 주지 않는다
2. 최적 부분 구조(Optimal Substructure)
: 문제에 대한 최종 해결 방법은 부분 문제에 대한 최적 문제 해결 방법으로 구성된다

### 완전 탐색 알고리즘(Brute-Force Algorithm)
완전 탐색 알고리즘은 무차별 대입 방법을 나타내는 알고리즘이다.<br>
순수한 컴퓨팅 성능에 의존하여 모든 가능성을 시도하여 문제를 해결하는 방법으로 공간복잡도와 시간복잡도의 요소를
고려하지 않고 최악의 시나리오를 취하더라도 솔루션을 찾으려고 하는 방법을 의미한다.<br>
따라서 완전 탐색 알고리즘은 최적의 솔루션이 아니라는 것을 의미하기도 한다.

### 이진 탐색 알고리즘(Binary Search Algorithm)
이진 탐색 알고리즘은 데이터가 정렬된 상태에서 절반씩 범위를 나눠 분할 정복기법으로 특정한 값을 찾아내는 알고리즘이다.
이진 탐색 알고리즘은 다음과 같은 단계로 동작한다.
1. 정렬된 배열의 가장 중간 인덱스를 지정한다
2. 찾으려고 하는 값이 지정한 중간 인덱스의 값이라면 탐색을 종료하고 아니면 다음 단계로 넘어간다
3. 찾으려고 하는 값이 중간 인덱스의 값보다 큰 값인지 작은 값은지 확인한다
4. 값이 있는 부분과 값이 없는 부분으로 분리한다
5. 값이 있는 부분에서 다시 1단계부터 반복한다
