## Spring Framework
### Spring Framework 모듈 구성
#### 아키텍처(Architecture)
- 시스템 구축 시 해당 시스템의 비즈니스적 요구 사항을 만족하는 전체 시스템 구조를 정의하는 것
- 이해 당사자들이 전체 시스템 구조를 이해할 수 있도록 이미지나 도형 등을 자주 사용
- 최대한 심플함을 유지하도록 노력

##### 시스템 아키텍처
- 하드웨어와 소프트웨어를 모두 포함하는 어떤 시스템의 전체적인 구성을 큰 그림으로 표현한 것

##### 소프트웨어 아키텍처
- 하드웨어를 제외한 컴퓨터 내의 모든 프로그램을 포관하는 의미를 지닌 소프트웨어의 구성을 큰 그림으로 표현한 것
- Ex) Java 플랫폼 아키텍처

##### 애플리케이션 아키텍처
- 계층형 아키텍처(N-티어)
  - API 계층(API Layer)
    - 클라이언트의 요청을 받아들이는 계층
  - 서비스 계층(Service Layer)
    - API 계층에서 전달 받은 요청을 업무 도메인의 요구 사항에 맞게 비즈니스적으로 처리하는 계층
      - 소프트웨어 공학의 도메인
      : 소프트웨어가 풀고자하는 현실 세상의 문제 해결 대상 분야 또는 해당 비즈니스 영역
    - 비즈니스 계층이라 불리기도 함
  - 데이터 액세스 계층(Data Access Layer)
    - 서비스 계층에서 처리된 데이터를 데이터베이스 등의 데이터 저장소에 저장하기 위한 계층

#### 아키텍처로 보는 Spring Framework 모듈 구성

![image](https://user-images.githubusercontent.com/105192204/195239497-52fbd726-9250-4523-a41f-54978634e468.png)

- Spring Framework에선 약 20여개의 모듈을 통해 다양한 기능을 제공
  - 모듈(Module)
    - Java에선 지원되는 여러가지 기능들을 목적에 맞게 그룹화하여 묶어 놓은 것을 의미
    - Java의 패키지 단위로 묶여 있으며, 패키지 안에는 관련 기능을 제공하기 위한 클래스들을 포함
    - 일반적으로 모듈을 재사용 가능하도록 라이브러리 형태로 제공되는 경우가 많음


## Spring Boot
- Spring 설정의 복잡함을 해결하기 위해 생겨난 Spring Project 중 하나
- Spring Boot를 사용해야하는 이유
  - XML 기반의 복잡한 설계 방식 지양
  - 의존 라이브러리의 자동 관리
  - 애플리케이션 설정의 자동 구성
  - 프로덕션급 애플리케이션의 손쉬운 빌드
  - 내장된 WAS를 통한 손쉬운 배포
    - WAS(Web Application Server)
      : 구현된 코드를 빌드해서 나온 결과물을 실제 웹 애플리케이션으로 실행되게 해주는 서버
